# BodyOS Constitution

## Mission

BodyOS helps the user make better daily decisions toward long-term body recomposition. It turns weight, food, sleep, steps, training, condition, alcohol, and coach notes into a credible operating view of the body.

## Vision

BodyOS is a Body Performance Operating System, not just a diet app. It should become the daily control panel for maintaining strength, reducing fat, recovering well, and staying consistent through real life.

## Product Goal

The product goal is to make the next good action obvious. BodyOS should help the user understand what happened today, what mattered, and what to adjust tomorrow.

## Target User

The primary user is a serious self-tracker pursuing body recomposition while balancing work, social events, fatigue, training, and imperfect days. The user wants useful coaching, not moral judgment.

## Core Principles

- Consistency beats perfection.
- Judge the process, not excuses.
- Measure, do not guess.
- Support realistic life, not idealized life.
- AI is a coach, not a judge.
- Data credibility is critical.
- The user should be able to continue using this daily.

## Coach Principles

- Be direct, practical, and supportive.
- Separate facts from interpretation.
- Reward recovery when recovery is the right choice.
- Treat event days as real life, not failure states.
- Prefer one clear next action over many abstract suggestions.
- Use data to create accountability without shame.
- Explain uncertainty when the data is incomplete or estimated.

## Product Principles

- Preserve trust before adding features.
- Make daily recording fast enough to sustain.
- Keep estimates realistic even when they are approximate.
- Prefer explicit user-provided values over inferred values.
- Maintain backward compatibility for existing records.
- Use documentation and PRDs before large behavior changes.
- Build the Streamlit version as a strong foundation before platform migration.
